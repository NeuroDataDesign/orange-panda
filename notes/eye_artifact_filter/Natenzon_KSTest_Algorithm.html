Describe Approach¶

1. Algorithm¶

Input:

  1. Theoritical cumulitive discribution of the function being tested. (In our case: The Mean across all electrodes) Numerical 1D Vector
  2. The Function we are testing Numerical 1D Vector

Output:

  1. KS Test Statistic (D,D-,D+) Float
  2. One (Or Two)- Tailed P-Value

Function Name

scipy.stats.kstest

Step-By-Step Instructions¶

$$ $$

1. Define the hypothesis of uniformity $$ $$ $$H_0: R_1 = U[0,1]$$ $$H_1: R_1 \neq U[0,1]$$ $$ $$ 2. Arrange data in increasing order $$ $$ $$R_1 \leq R_2 \leq ... \leq R_n$$ $$ $$ 3. Compute $D^+$ and $D^-$ $$ $$ $$D^+ = max (\frac{i}{N}-R_i), 1 \leq i \leq N$$ $$ $$ $$D^- = max (R_i-\frac{i-1}{N}), 1 \leq i \leq N$$ $$ $$ 4. Compute D $$ $$ $$ D = max(D^+,D^-) $$ $$ $$ 5. Compute Critical Value based on $\alpha$ provided $$ $$ $$D_\alpha$$ $$ $$ 6. Compare $$ $$ $$ D>D_\alpha $$ H_0 hypothesis is rejected. This means H0 values are not uniform $$ D<D\alpha $$ H_0 hypothesis is accepted. This means H_0 values are uniform.

2. Simulation¶

Advantages

  1. Does not depend on the udnerlying cumulative distribution function being tested

Limitations

  1. Only applies to continuous distributions
  2. More sensitive near the center of the distribution than the tails
  3. Location, scale, and shape of distribution must be specified to identify the critical region for which the K-S test is valid.

1. A simulation setting for which the alg with perform very well

The algorithm will work well with a continuous, normally distribution. $$ $$ 2. A simulation setting for which the alg will not perform well

The algorithm will not work well for a non-normal / non-continuous data set.

3. Analysis¶

1. Qualitative visualizations per trial for qualitatively evaluating performance

After calculating the K-S test statistic (D) for each independent component found by ICA to the mean of all components, we can identify which components had the largest D values. We can then visually examine these components (suspected artifacts) by plotting them, as well as by plotting the original source signal with each of the suspected artifact components removed. From this, we can qualitatively evaluate the performance by determining if the artifact components (those with largest D values) have a similar shape to Eye/EoG artifacts.

2. Quantitative "score" per trial for quantitatively evaluating performance

We cannot score the per trial for quantitatively evaluating performance with real data. However, we can test the algorithm quantitatively by generating 1000 random numbers for a:

  1. Normal Distribution
  2. Double Exponential Distribution
  3. t (w/ 3 degrees of freedom) Distribution
  4. Log-Normal Distribution

We identify the significance level to be $\alpha$ = 0.05, which leads to a critical value of 0.04301.

We then reject the distributions with a D value greater than the critical value of 0.04301. In theory, we should accept the 1st (normal) distribution, and reject other three.

We quantitatively evaluate the performance by running the KS test on "X" number of randomly generated distributions (with "Y" number of normal distributions). Each time the distribution passes the KS test, we add 1 to variable z (starts at 0). After running the KS test on X number of distributions, we calculate the percentage of expected accepted distributions ($\frac{Y}{X} \times 100)$) and compare it to the actual accepted distributions ($\frac{Z}{X} \times 100)$). We then calculate the percentage deviation from the expected, accepted distributions by: $$\frac{Expected - Actual}{Expected}$$

3. Summary Plot visualization of quantitative scores over all trials (often a histogram)

We can plot a summary of the quantitative scores over all trials using a histogram. Where the x-axis represents: various ranges for percentage deviation of the resulting number of accepted distributions to the expected. The y-axis represents: the quantity of simulations that fall within a given range of percent deviation. Each histogram should represent the results for one distinct number of randomly generated distributions. Therefore, multiple histograms must be used if we were to test the KS test with varying numbers of total (X) distributions.
